				<h1 class="entry-title">Developer Information</h1>
			</header><!-- .entry-header -->

			<div class="entry-content">
				<p>
					You&#8217;ve got a cool new plugin and are hoping to give it some exposure. You&#8217;re in the right place. Just <a href="https://wordpress.org/plugins/developers/add/">ask us to host it for you</a>. You&#8217;ll be able to:				</p>
				<ul>
					<li>Keep track of how many people have downloaded it.</li>
					<li>Let people leave comments about your plugin.</li>
					<li>Get your plugin rated against all the other cool WordPress plugins.</li>
					<li>Give your plugin lots of exposure in this centralized repository.</li>
				</ul>

				<h3>There are some restrictions</h3>
				<ul>
					<li>
						Your plugin must be compatible with the <a href="https://www.gnu.org/licenses/license-list.html#GPLCompatibleLicenses">GNU General Public License v2</a>, or any later version. We strongly recommend using the same license as WordPress — &#8220;GPLv2 or later.&#8221;					</li>
					<li>The plugin must not do anything illegal or be morally offensive (that&#8217;s subjective, we know).</li>
					<li>
						You have to actually use the <a href="https://subversion.apache.org/">Subversion</a> repository we give you in order for your plugin to show up on this site. The WordPress Plugin Directory is a hosting site, not a listing site.					</li>
					<li>The plugin must not embed external links on the public site (like a &#8220;powered by&#8221; link) without explicitly asking the user&#8217;s permission.</li>
					<li>
						Your plugin must abide by our list of <a href="https://developer.wordpress.org/plugins/wordpress-org/detailed-plugin-guidelines/">detailed guidelines</a>, which include not being a spammer and not abusing the systems.					</li>
				</ul>

				<h3>Submission is Simple</h3>
				<ol>
					<li>
						<a href="https://login.wordpress.org/register?locale=en_US">Sign up</a> for an account on WordPress.org.					</li>
					<li>
						<a href="https://wordpress.org/plugins/developers/add/">Submit your plugin for review</a>.					</li>
					<li>
						After your plugin is <a href="https://developer.wordpress.org/plugins/wordpress-org/plugin-developer-faq/#questions-about-submissions-and-approval">manually reviewed</a>, it will either be approved or you will be emailed and asked to provide more information and/or make corrections.					</li>
					<li>
						Once approved, you&#8217;ll be given access to a <a id="subversion" href="https://developer.wordpress.org/plugins/wordpress-org/how-to-use-subversion/">Subversion Repository</a> where you&#8217;ll store your plugin.					</li>
					<li>
						Shortly after you upload your plugin (and a <a href="#readme">readme file</a>!) to that repository, it will be automatically displayed in the <a href="https://wordpress.org/plugins/">plugins browser</a>.					</li>
					<li>
						Check out the <strong><a href="https://developer.wordpress.org/plugins/wordpress-org/plugin-developer-faq/">FAQ</a></strong> for more information.					</li>
				</ol>

				<h3 id="readme">Readme files</h3>
				<p>
					To make your entry in the plugin browser most useful, each plugin should have a readme file named <code>readme.txt</code> that adheres to the <a href="https://wordpress.org/plugins/readme.txt">WordPress plugin readme file standard</a>. You can put your readme file through the <a href="https://wordpress.org/plugins/developers/readme-validator/">readme validator</a> to check it.				</p>
